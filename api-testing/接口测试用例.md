# 接口测试用例

## 用例字段

| 字段 | 说明 |
| --- | --- |
| 用例编号 | `API_模块_编号` |
| 模块 | 接口所属模块 |
| 优先级 | P0 核心流程 |
| 接口 | 请求方法和接口路径 |
| 验证点 | 业务预期和执行结果 |

## 用例说明

本轮 Postman 共执行 20 条请求，覆盖 18 条核心接口用例。第 15 条请求用于获取 `adoptionRequestId`，第 17/18 条同属 `API_COMM_001` 沟通消息接口验证。

## 认证与用户

| 用例编号 | 模块 | 接口 | 验证点 |
| --- | --- | --- | --- |
| API_HEALTH_001 | 健康检查 | GET `/api/health` | 服务健康检查成功，返回 `code=0` |
| API_AUTH_001 | 认证 | POST `/api/auth/register` | 普通用户注册成功 |
| API_AUTH_002 | 认证 | POST `/api/auth/login` | 普通用户登录成功，返回 token 和用户信息 |
| API_AUTH_003 | 认证 | POST `/api/auth/login` | 管理员登录成功，返回管理员 token |
| API_AUTH_004 | 认证 | GET `/api/auth/me` | 携带 Token 获取当前用户信息成功 |
| API_USER_001 | 用户 | PUT `/api/users/profile` | 修改个人资料成功 |

## 救助流程

| 用例编号 | 模块 | 接口 | 验证点 |
| --- | --- | --- | --- |
| API_RESCUE_001 | 救助广场 | POST `/api/rescues` | 普通用户发布救助信息成功，保存 `rescueId` |
| API_ADMIN_RESCUE_001 | 管理员救助 | POST `/api/admin/rescues/{id}/approve` | 管理员审核通过救助信息 |
| API_RESCUE_002 | 救助广场 | GET `/api/rescues` | 查询救助广场列表成功 |
| API_COMMENT_001 | 评论 | POST `/api/rescues/{id}/comments` | 用户发表评论成功 |
| API_ADMIN_RESCUE_002 | 管理员救助 | POST `/api/admin/rescues/{id}/adopt` | 管理员收编救助信息成功 |

## 领养流程

| 用例编号 | 模块 | 接口 | 验证点 |
| --- | --- | --- | --- |
