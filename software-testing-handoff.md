# 软件测试求职准备交接文档

## 1. 当前目标

我的目标不是继续大量刷软件测试教学视频，而是更高效地完成下面几件事：

1. 基于自己真实部署并跑通过的毕业设计项目，整理出一个可信的软件测试项目案例。
2. 围绕这个项目完成接口测试、功能测试、基础性能测试、自动化回归脚本等材料。
3. 根据测试岗位 JD 定制简历项目经历，提高简历和岗位的匹配度。
4. 针对简历里的项目准备面试题和项目追问，而不是孤立背概念。
5. 尽快开始投递，在投递和面试反馈中继续补短板。

核心原则：

> 不要等“学完所有测试知识”再投简历。应届生阶段更重要的是证明自己能上手、能执行、能讲清楚一个真实项目。

## 3. 项目是否适合作为测试案例

结论：适合，而且比网上很多通用商城案例更适合作为我的主项目。

原因：

1. 这是我自己的毕业设计项目，面试时可信度高。
2. 业务不是简单 CRUD，有用户、管理员、状态流转、文件上传、评论、消息沟通等场景。
3. 适合做接口测试、功能测试、权限测试、异常测试、基础性能测试和自动化回归。
4. 业务流程可以讲清楚，能支撑面试官追问。

适合包装成：

```text
流浪动物救助与领养平台测试项目
```

简历中可以强调：

```text
独立搭建本地测试环境，基于真实项目完成测试计划、功能测试、接口测试、权限测试、基础性能测试和接口自动化回归。
```

## 4. 现在最需要做什么

当前最重要的不是继续刷视频，而是按下面顺序产出真实材料。

### 第一优先级：整理接口测试案例

接口测试最适合先做，因为项目后端接口完整，业务流清晰，也最容易形成作品集。

建议工具：

```text
Apifox 或 Postman
```

要整理的接口模块：

```text
认证模块：注册、登录、获取当前用户
用户模块：修改资料、上传头像
救助模块：发布救助、列表、详情、我的救助、删除
评论模块：查询评论、发表评论、删除评论
管理员救助模块：待审核列表、全部列表、审核通过、驳回、收编、删除
领养宠物模块：宠物列表、详情、管理员新增、修改、删除
领养申请模块：用户申请、我的申请、管理员查看、进入沟通、通过、驳回、签收、删除
沟通模块：查询消息、发送消息
```

接口测试重点：

```text
正常请求
参数为空
参数非法
Token 缺失
Token 错误
普通用户访问管理员接口
非本人操作数据
重复提交
错误状态流转
文件上传异常
```

接口测试产出物：

```text
接口测试用例表
Postman/Apifox 接口集合
环境变量配置：baseUrl、userToken、adminToken
接口断言：code、message、data、状态流转
接口测试报告
```

### 第四优先级：做基础性能测试

性能测试可以做，但不要夸大成大型压测。

合理定位：

```text
对核心接口做基础并发测试，观察响应时间、错误率和吞吐量。
```

建议工具：

```text
JMeter
```

优先压测接口：

```text
GET /api/health
POST /api/auth/login
GET /api/rescues
GET /api/adoption/pets
POST /api/adoption/requests
```

建议并发场景：

```text
20 并发，持续 1 分钟
50 并发，持续 3 分钟
100 并发，持续 5 分钟
```

观察指标：

```text
平均响应时间
P90/P95 响应时间
吞吐量 TPS
错误率
最大响应时间
服务端日志是否报错
数据库是否出现明显慢查询
```

性能测试报告应包括：

```text
测试目的
测试环境
测试接口
并发模型
测试数据
测试结果
问题分析
优化建议
```

## 8. 网上实战项目如何使用

可以把网上商城、小程序、后台管理系统等测试实战资料发给 AI 分析，但不要直接照搬成自己的经历。

正确用法：

```text
借鉴测试点
借鉴用例模板
借鉴缺陷描述
借鉴接口测试结构
借鉴性能测试报告格式
借鉴简历表达方式
```

不建议：

```text
把没有跑过的项目写成自己完整做过
把培训项目原封不动写进简历
写自己无法解释的技术点
写自己没有执行过的复杂自动化框架
```

更好的做法：

```text
用网上案例补充思路，再迁移到自己的宠物救助领养项目中。
```

例如商城订单状态流转可以迁移为：

```text
领养申请状态：待审核 -> 沟通中 -> 运输中 -> 已签收
救助记录状态：待审核 -> 展示中 -> 已收编 / 已驳回
```

## 9. 面试准备方式

不要孤立背题，要围绕自己的项目准备。

必须准备的问题：

```text
你这个项目是做什么的？
你为什么选择这个项目做测试？
你负责测试哪些模块？
你设计了哪些测试用例？
接口测试怎么做的？
Token 鉴权怎么测？
普通用户和管理员权限怎么测？
你发现过哪些 bug？
你怎么验证数据库状态？
性能测试压了哪些接口？
自动化回归脚本怎么写的？
如果接口依赖前一个接口返回值，你怎么处理？
如果测试环境数据乱了，你怎么处理？
如果开发说这不是 bug，你怎么办？
```

回答原则：

```text
先说测试思路
再说项目里的具体例子
最后说验证结果
```

示例：

```text
我在领养申请接口中主要测了正常申请、重复申请、未登录申请、宠物不存在、宠物已被领养、普通用户访问管理员审核接口等场景。接口测试时用环境变量保存 userToken 和 adminToken，通过断言校验 code、message 和状态变化，并在 MySQL 中检查 adoption_request 表的状态是否符合预期。
```

这样比单纯背“什么是接口测试”更有说服力。

## 12. 关键提醒

1. 可以借鉴网上案例，但简历里写的内容必须能讲清楚。
2. 不要陷入“学完所有工具再投递”的误区。
3. 应届生不需要装成资深测试工程师，重点是能证明自己有项目、有方法、有执行力。
4. 这个宠物救助领养项目是主项目，网上项目只是补充素材。
5. 简历要根据 JD 定制，不要一份简历投所有岗位。
6. 面试题要结合项目背，不能只背概念。
7. 最快的成长方式是：产出材料、投递、面试、复盘、再优化。
