# 项目面试话术

## 1 分钟项目介绍

我用自己的毕业设计项目做了一个软件测试作品集。这个项目是流浪动物救助与领养平台，前端使用 Vue3，后端使用 Spring Boot，数据库是 MySQL，整体是前后端分离架构。

业务上主要有两条线：第一条是救助广场，普通用户发布流浪动物救助信息，管理员审核，通过后展示到救助广场，用户可以评论，管理员也可以收编；第二条是领养中心，管理员发布宠物档案，普通用户申请领养，管理员进入沟通并审核，通过后用户确认签收。

我围绕这两条业务线做了功能测试、接口测试、权限测试、基础性能测试和接口自动化回归。接口测试使用 Postman 配置环境变量和 Token，自动提取登录返回的 token，并对核心接口添加状态码、业务 code 和关键字段断言。自动化回归方面，我用 Shell + curl 覆盖了注册登录、救助审核、评论、收编、领养申请、沟通、审核、签收以及驳回分支。性能测试方面，我用 JMeter 对登录、救助列表、领养列表等接口做了基础并发测试，关注响应时间、吞吐量和错误率。

## 高频追问

### 你为什么选这个项目做测试？

因为这是我自己真实开发和部署过的毕业设计项目，业务流程完整，不只是简单 CRUD。它包含用户和管理员两类角色，也有审核、状态流转、文件上传、评论、沟通消息、权限控制等场景，适合做功能测试、接口测试、权限测试和基础性能测试。

### 接口测试怎么做的？

我先根据后端 Controller 和需求文档整理接口清单，再按模块设计测试用例。单接口先测正常请求，再测异常参数、缺参、未登录和权限问题。业务场景用例主要覆盖救助广场和领养中心两条主流程。Postman 里配置了 `baseUrl`、`userToken`、`adminToken` 等环境变量，登录后通过 Tests 脚本提取 token，后续接口通过请求头携带 token。

### Token 鉴权怎么测？

我主要测三类：第一是携带正确 Token 可以访问自己的接口；第二是不带 Token 或错误 Token 访问受保护接口应失败；第三是普通用户携带自己的 Token 访问管理员接口应返回权限不足。

### 你设计了哪些核心用例？

核心用例包括：用户注册登录、发布救助信息、管理员审核通过、救助广场展示、发表评论、管理员收编；领养中心包括管理员发布宠物、用户申请领养、管理员进入沟通、双方发送消息、管理员审核通过、用户确认签收。另外还设计了驳回分支、重复申请、未登录申请、普通用户访问管理员接口等异常用例。

### 自动化回归脚本怎么写的？

我用 Shell + curl 写了接口回归脚本。脚本先注册新用户并登录，提取用户 Token 和管理员 Token，然后按业务顺序调用接口。比如救助主线是发布救助、审核通过、评论、收编，领养主线是发布宠物、申请领养、进入沟通、发送消息、审核通过、签收。脚本会校验响应里的业务 code，如果接口失败会输出响应内容并中断。

### 性能测试怎么做的？

我用 JMeter 做基础并发测试，不做夸大的大型压测。主要选择健康检查、登录、救助列表、领养宠物列表这些核心接口，设计 20、50、100 并发的小规模场景，观察平均响应时间、P95 响应时间、吞吐量和错误率。因为测试是在本地环境执行，所以结论只作为基础性能评估，不代表生产容量。

### 如果开发说这不是 Bug，你怎么办？

我会先回到需求文档和产品规则，看实际结果是否和需求冲突。如果需求明确，我会用复现步骤、接口响应、页面截图或数据库状态说明问题。如果需求不明确，我会找产品确认预期，再决定是缺陷、需求变更还是优化建议。
