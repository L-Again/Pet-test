# Pet-test

流浪动物救助与领养平台测试项目。

本仓库展示一个前后端分离 Web 项目的测试过程与交付物，测试对象包含用户注册登录、救助信息发布与审核、救助评论、管理员收编、领养宠物发布、领养申请、沟通审核、用户签收和角色权限控制等业务流程。

## 项目概况

| 项目 | 内容 |
| --- | --- |
| 系统类型 | 宠物救助与领养平台 |
| 前端技术 | Vue 3、Vite、Element Plus、Axios |
| 后端技术 | Spring Boot、Java 17、MyBatis-Plus |
| 数据库 | MySQL |
| 接口风格 | REST API |
| 鉴权方式 | JWT Token |

## 测试内容

| 测试类型 | 覆盖内容 | 主要交付物 |
| --- | --- | --- |
| 接口测试 | 注册登录、救助发布审核、评论、收编、领养申请、沟通、审核通过、签收闭环 | Postman Collection、接口用例、接口执行报告、回归脚本说明 |
| 功能测试 | 普通用户和管理员两类角色的核心页面流程、状态变化和权限入口 | 功能测试用例、功能测试执行报告、页面截图 |
| 性能测试 | 健康检查、登录、救助列表、领养列表核心接口基础并发测试 | JMeter 测试计划、性能测试方案与执行报告、聚合报告截图 |

## 仓库结构

## 核心结论

本轮测试围绕核心业务闭环展开：

```text
用户发布救助 -> 管理员审核 -> 救助广场展示 -> 用户评论 -> 管理员收编
管理员发布领养宠物 -> 用户申请 -> 管理员沟通审核 -> 用户签收
```

执行结果：

| 类型 | 结果 |
| --- | --- |
| 接口测试 | Postman 执行 20 条请求，对应 18 条核心接口用例，全部通过 |
| 接口回归 | Shell + curl 回归脚本执行通过 |
| 功能测试 | 17 条功能用例执行通过 |
| 性能测试 | 20、50、100 并发场景均完成，整体错误率 0.00% |
| 缺陷记录 | 本轮未发现稳定可复现缺陷，已记录测试风险 |

## 说明

本仓库仅保留脱敏后的测试交付物。Postman 本地环境文件、Token、明文密码、本机绝对路径、数据库配置、JMeter 原始 JTL 文件和自动生成 HTML 报告不纳入公开仓库。
