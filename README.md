# Pet-test

流浪动物救助与领养平台测试项目。

本仓库用于展示一个前后端分离 Web 项目的测试过程与交付物。被测系统包含用户注册登录、救助信息发布与审核、救助评论、管理员收编、领养宠物发布、领养申请、沟通审核、用户签收和角色权限控制等业务流程。

## 项目概况

| 项目 | 内容 |
| --- | --- |
| 系统类型 | 宠物救助与领养平台 |
| 前端技术 | Vue 3、Vite、Element Plus、Axios |
| 后端技术 | Spring Boot、Java 17、MyBatis-Plus |
| 数据库 | MySQL |
| 接口风格 | REST API |
| 鉴权方式 | JWT Token |

## 测试范围

| 模块 | 测试重点 |
| --- | --- |
| 认证与用户 | 注册、登录、Token 鉴权、个人资料修改 |
| 救助广场 | 救助信息发布、管理员审核、列表展示、详情查看、用户删除 |
| 救助评论 | 评论发布、评论查询、评论删除 |
| 管理员救助 | 待审核列表、审核通过、审核驳回、收编、后台删除 |
| 领养中心 | 宠物列表、宠物详情、管理员发布宠物档案 |
| 领养申请 | 用户申请、管理员沟通、审核通过、审核驳回、用户签收 |
| 权限控制 | 未登录访问、普通用户访问管理员接口、非本人数据操作 |
| 性能测试 | 登录、救助列表、领养列表等核心接口的基础并发测试 |

## 核心测试思路

本项目按照“需求分析 -> 测试计划 -> 用例设计 -> 测试执行 -> 缺陷记录 -> 测试报告”的流程展开。

接口测试优先覆盖核心业务链路和状态流转，包括救助发布审核、评论收编、领养申请、沟通审核、签收闭环等场景。功能测试围绕普通用户和管理员两类角色展开，重点验证页面流程、权限隔离和异常提示。性能测试定位为基础并发测试，关注核心接口的响应时间、吞吐量和错误率。

## 自动化回归

项目提供 Shell + curl 形式的接口回归脚本说明，用于验证注册登录、救助审核、评论、收编、领养申请、沟通、审核通过、签收以及驳回分支等核心接口流程。
