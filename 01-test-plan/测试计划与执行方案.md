## 执行安排

### 第 1 阶段：测试准备

1. 确认后端、前端和 MySQL 可正常启动。
2. 确认管理员账号和普通用户测试账号可用。
3. 准备测试图片、测试手机号、测试用户名等基础数据。
4. 确认测试过程中不使用生产数据。

### 第 2 阶段：接口测试

1. 启动后端服务，确认 `GET /api/health` 正常。
2. 在 Postman 创建环境变量：
   - `baseUrl=http://localhost:8080`
   - `userToken`
   - `adminToken`
   - `rescueId`
   - `petId`
   - `adoptionRequestId`
3. 按 `02-test-cases/接口测试用例.md` 执行核心用例。
4. 登录接口提取 Token，后续接口使用 `Authorization: Bearer {{userToken}}` 或 `Authorization: Bearer {{adminToken}}`。
5. 每个关键接口至少添加：
   - HTTP 状态码断言
   - 业务 `code` 断言
   - 核心字段存在断言
6. 导出 Collection，放入 `postman/`。
7. 可选：使用 Newman 生成 HTML 报告，放入 `reports/newman/`。

### 第 3 阶段：自动化回归脚本

1. 启动后端。
2. 运行：

```bash
cd /Users/nolan/Developer/code/Java_code/Pet/backend/backend
./scripts/full_flow_test.sh
```

3. 保存终端执行结果截图或复制关键输出到 `04-regression-script/执行结果记录.md`。
4. 如果失败，记录失败接口、响应内容和定位过程。

### 第 4 阶段：功能测试

1. 启动前端和后端。
2. 按普通用户和管理员两种角色执行页面流程。
3. 优先执行：
   - 注册登录
   - 修改个人资料
   - 发布救助
   - 管理员审核救助
   - 救助评论
   - 管理员收编
   - 管理员发布领养宠物
   - 用户申请领养
   - 管理员沟通与审核
   - 用户签收
   - 权限控制
4. 实际结果写回功能测试用例表。
5. 发现问题写入缺陷记录表。

### 第 5 阶段：性能测试

1. 使用 JMeter 创建线程组。
2. 先压测 `GET /api/health`，确认工具和服务可用。
3. 再压测：
   - 登录接口
   - 救助列表接口
   - 领养宠物列表接口
4. 建议场景：
   - 20 并发，持续 1 分钟
   - 50 并发，持续 2 分钟
   - 100 并发，持续 3 分钟
5. 记录平均响应时间、P90/P95、吞吐量、错误率。
6. 输出基础性能测试结论。

### 第 6 阶段：测试结果整理

1. 回填功能测试用例和接口测试用例的实际结果。
2. 整理 Postman/Newman 执行结果。
3. 整理自动化回归脚本执行结果。
4. 整理 JMeter 聚合报告关键指标。
5. 记录缺陷和测试风险。
