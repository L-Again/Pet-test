# 项目背景与测试范围

## 项目概述

宠物领养与救助平台是一个前后端分离的公益类 Web 项目，围绕流浪动物从用户发现、发布救助、管理员审核、救助广场展示、官方收编，到领养中心建档、用户申请领养、管理员沟通审核、用户签收的闭环流程展开。

## 用户角色

| 角色 | 主要权限 |
| --- | --- |
| 普通用户 | 注册登录、修改个人资料、发布救助信息、评论救助信息、申请领养、发送沟通消息、确认签收、删除自己的记录 |
| 管理员 | 登录后台、审核救助信息、驳回救助信息、收编救助动物、发布领养宠物、审核领养申请、删除后台记录 |

## 核心业务流程

### 救助广场

```text
用户发布救助信息 -> 管理员审核通过 -> 救助广场展示 -> 用户评论 -> 管理员收编 -> 用户/管理员删除记录
```

分支流程：

```text
用户发布救助信息 -> 管理员驳回 -> 用户删除记录
```

### 领养中心

```text
管理员发布宠物档案 -> 用户申请领养 -> 管理员进入沟通 -> 双方发送消息 -> 管理员审核通过 -> 用户签收 -> 用户/管理员删除记录
```

分支流程：

```text
管理员发布宠物档案 -> 用户申请领养 -> 管理员进入沟通 -> 管理员驳回 -> 用户删除记录
```

## 主要接口范围

| 模块 | 接口范围 |
| --- | --- |
| 健康检查 | `GET /api/health` |
| 认证 | 注册、登录、获取当前用户 |
| 用户 | 修改资料、上传头像 |
| 救助广场 | 发布、列表、详情、我的发布、用户删除 |
| 救助评论 | 查询评论、发表评论、删除评论 |
| 管理员救助 | 待审核列表、全部列表、通过、驳回、收编、管理员删除 |
| 领养宠物 | 用户列表、详情、管理员新增、修改、删除 |
| 领养申请 | 用户申请、我的申请、管理员查看、进入沟通、通过、驳回、签收、删除 |
| 沟通消息 | 查询消息、发送消息 |

## 测试重点

- 核心主流程是否能跑通。
- 分支流程是否符合状态机。
- JWT 鉴权是否生效。
- 普通用户页面是否隐藏管理入口。
- 未登录用户是否被引导到登录页。
- 核心接口在基础并发下的响应时间、吞吐量和错误率。
